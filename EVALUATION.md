# 타이포그래피 개선 평가 로그 (PINK × BLACK)

- 대상: index.html (한국 공공데이터 MCP 허브)
- 방식: BLACK(generator)이 만들고 PINK(UI/UX 전문가)가 10점 만점 평가, 9.5 합격선
- 날짜: 2026-07-16

## 폰트 시스템 (최종)

| 역할 | 폰트 | 사용처 |
|---|---|---|
| 한글 전체 | Pretendard Variable | 본문, UI, 세리프 요소의 한글 폴백 |
| 영문 디스플레이 | Newsreader (serif) | 로고, 헤드라인, 섹션 제목, 카운터 숫자, 키캡 |
| 영문 본문·UI | Instrument Sans | 설명문, 버튼, 배지, 네비게이션의 라틴 글자 |
| 데이터·코드 | IBM Plex Mono | 저장소 이름, ★/⑂ 수치, 칩 카운트, `/` 단축키 힌트 |

원칙: **"저장소 이름과 수치는 모노, 문장은 산세리프, 표제는 세리프."**
한글은 모든 스택에서 Pretendard로 폴백된다 (라틴 폰트를 스택 앞에 두는 방식).

## ROUND 1 — 8.9 / 10 (불합격)

BLACK 작업: Instrument Sans + IBM Plex Mono 도입, 카드 이름 모노화(14px).

PINK 지적:
1. 카드 이름 14px가 50px 아이콘 대비 약해 위계 흔들림 → 15px로
2. 카드 ★/⑂ 수치만 산세리프 — 칩 카운트·kbd는 모노라 시스템 일관성 붕괴 → Plex Mono 통일
3. 모노에 letter-spacing -.02em 과도 → -.01em, 이름-배지 간격 5→6px

## ROUND 2 — 9.6 / 10 (합격)

BLACK 작업: 위 3건 반영.

PINK 확인: 위계 9.5, 일관성 9.8, 디테일 9.5. "수치는 모노" 규칙이 시스템으로 성립.

## 11차: 약자 풀어쓰기 + 초기화 버그 수정 (2026-07-17)

### 버그: 초기화를 눌러도 결과 스트립이 계속 표시

원인: `.result-info{display:flex}`(저자 스타일)가 UA의 `[hidden]{display:none}`을 덮어씀 — 저자 스타일은 UA 스타일보다 항상 우선. `hidden` 속성은 정상적으로 토글되고 있었으나 시각적으로만 계속 보였다. to-top 버튼에는 `[hidden]` override를 넣었으면서 이 스트립에는 빠뜨림. `.result-info[hidden]{display:none}` 추가로 해결. computed display 실측: 초기 none → 검색 flex → 초기화 none.
