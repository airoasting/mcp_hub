# AGENTS.md — MCP 허브 운영 규칙

이 문서는 이 폴더에서 작업하는 모든 AI 에이전트와 사람이 따르는 단일 기준이다. 규칙이 서로 어긋나면 이 문서가 이긴다. 현재 상태와 결정 이력은 `MEMORY.md`, 품질 평가 로그는 `EVALUATION.md`에 있다.

## 1. 프로젝트 정의

- **산출물**: `index.html` 하나. 외부 파일 의존 없는 단일 HTML (Google Fonts CDN만 허용).
- **독자**: 한국 비즈니스 리더. 개발자가 아니다. 모든 표기·선정 판단의 기준점.
- **정본 데이터**: `index.html` 안의 `DATA` 배열. 별도 DB나 JSON 없음. 카드 수·카테고리 수는 이 배열에서 자동 계산되므로 손으로 세지 않는다.
- **목적**: 커넥터를 "보자마자 이해하고, 눌러서 GitHub 저장소로 이동"하게 하는 카탈로그. 다른 기능을 붙이지 않는다.

## 2. 카드 선정 기준

새 카드를 넣거나 기존 카드를 지울 때 아래 6개 기준을 순서대로 통과시킨다. 하나라도 걸리면 등재하지 않는다.

### 기준 1. MCP 서버만 등재한다

- MCP 프로토콜 서버 구현체만 넣는다. 판별법: 저장소에 MCP SDK 의존성, stdio/HTTP 서버 엔트리포인트, tools 스키마 선언이 있어야 한다. `SKILL.md`와 스크립트만 있으면 스킬이지 MCP가 아니다.
- 모음집·디렉터리(awesome 리스트, 공식 servers 컬렉션)는 서버가 아니므로 넣지 않는다.
- MCP를 포함한 이중 인터페이스(스킬+MCP 동시 제공)는 허용.
- **예외 A**: NomaDamas k-skill 중 "사용자 로그인 필요" 스킬. 타 사이트 로그인 세션 기반 자동화라는 특수성으로 등재. 스타·포크는 모음집 저장소 전체 값이며 링크는 각 스킬 폴더로 연결한다.
- **예외 B**: airoasting 자사 스킬(현재 dart 1건). 자사 제품 선례. 추가하려면 사용자 승인이 필요하다.

### 기준 2. 출처 직관성 — 이 카탈로그의 고유 기준

출처 배지를 **보자마자** 한국 비즈니스 리더가 인지하지 못하는 서비스는 스타가 아무리 많아도 넣지 않는다. 설명이 필요한 출처는 이미 실격이다.

- 탈락 사례: ODsay, Exa, Tavily, Kagi, Brave, Financial Datasets, Massive(구 Polygon), Alpaca, Equibles, CourtListener, "US 법률"(출처 표기 자체가 모호).
- 경계선 판정법 두 가지. ① 이름에서 뜻이 보이면 유지(숲나들e, 도서관정보나루). ② 글로벌 브랜드 인지도가 충분하면 유지(SEC EDGAR, Zillow, AccuWeather, arXiv, NotebookLM, HubSpot, 팝빌, 두레이).
- 판단이 애매하면 사용자에게 묻는다.

### 기준 3. 비즈니스 리더 대상

개발자 전용 도구는 넣지 않는다. 클라우드 인프라, DB, 코딩 에이전트, 코드 검색, 토큰 최적화류가 해당한다. 브랜드가 유명해도(예: GitHub MCP Server) 용도가 개발자향이면 보류하고 사용자에게 묻는다.

### 기준 4. 같은 출처는 스타 최다 1개

같은 데이터 출처(법제처, 기상청, 금융감독원 등)를 다루는 저장소가 여럿이면 스타 최다 1개만 등재한다. 더 높은 스타의 저장소가 나타나면 교체한다(예: 국가데이터처 소스를 ★12에서 ★76으로 교체한 선례).

### 기준 5. 링크 유효성

404, 비공개, 아카이브된 저장소는 넣지 않는다. 발견 즉시 제거하고 `MEMORY.md`에 기록한다.

### 기준 6. MECE 분류

- 카테고리는 업무 도메인 단일 축 10개: 법률·특허 / 금융·투자 / 경제·통계 / 부동산 / 지도·교통 / 검색·리서치 / 생활·여행 / 채용·HR / 영업·CRM / 문서·협업.
- 지역(한국/미국)은 카테고리가 아니라 출처 배지로 표현한다.
- 특정 도메인(법률, 부동산 등)이 있으면 그 도메인으로. 범용 포털·통계는 경제·통계로. 날씨는 생활·여행으로.

## 3. 표기 원칙

- **설명 문구는 약자를 풀어 쓴다**. DART→금융감독원 전자공시, KRX→한국거래소, KOSIS→국가통계포털, ECOS→경제통계시스템, HIRA→건강보험심사평가원, EDGAR→미국 증권거래위원회 전자공시, FRED→미국 연방준비제도. 풀어도 도움이 안 되는 전문용어(XBRL)는 지운다.
- **출처 배지는 브랜드명 유지** (SEC EDGAR, Yahoo Finance). 배지는 인지용, 설명은 이해용으로 역할이 다르다.
- 스타·포크는 GitHub API 실측값만 쓰고, 푸터에 기준일을 명시한다. 지어내지 않는다.
- 인증 구분은 3종: 바로 사용(none) / 로그인만 필요(login) / API 발급 필요(api).

## 4. 변경 절차

### 카드 추가

1. 기준 1→6 순서로 통과 확인.
2. GitHub API로 스타·포크 실측 (`curl -s https://api.github.com/repos/{repo}`).
3. `DATA` 배열의 해당 카테고리 블록에 삽입. 형식: `[name, 이니셜, 카테고리, 배지, stars, forks, desc, repo경로, auth]`.
4. 아래 검증 체크리스트 실행.

### 카드 삭제

삭제는 카드만 지우고 끝나지 않는다. 그 카드를 참조하던 것들을 역추적한다.

1. 히어로 카피가 그 카드를 근거로 하는지 (예: "미국 판례까지" 문구는 CourtListener 삭제 때 함께 수정했다).
2. 퀵픽 키워드가 그 카드 때문에 유지되는지 (예: "판례" 퀵픽이 1건이 되어 "공시"로 교체했다).
3. `MEMORY.md`의 삭제 기록에 사유와 함께 추가.

### 기준과 충돌하는 요청

사용자가 기준과 어긋나는 요청을 하면 침묵 속에 따르지도, 거부하지도 않는다. 충돌 표를 만들어 보여주고 결정권을 넘긴다. 특히 사용자가 과거에 직접 삭제를 지시한 카드는 어떤 명분으로도 조용히 부활시키지 않는다.

## 5. 품질 기준

- 수정 작업은 BLACK(제작)/RED(평가) 루프로 진행한다. 합격선 9.5/10, 어느 차원도 9.0 미만이 없어야 한다. 차원과 가중치: 기능·정확성 30% / UX·인터랙션 20% / 비주얼·완성도 20% / 코드 견고성 15% / 반응형·접근성 10% / 디테일·마감 5%.
- 평가는 `EVALUATION.md`에 append. 과거 라운드를 지우지 않는다.

### 검증 체크리스트 (수정 후 필수)

- [ ] 콘솔 에러 0
- [ ] 가로 오버플로 없음: 375 / 760 / 1280px
- [ ] 카운터 숫자 = 카드 수 = 카테고리 합계
- [ ] 검색 회귀: 출처 검색("토스" 1건 이상), 빈 상태, 초기화
- [ ] 퀵픽 토글 on/off, 칩 스크롤스파이
- [ ] 시각 상태는 computed style로 검증 (hidden 속성만 믿지 않는다)

### 코드 함정 (반복 실수 방지)

- `display`를 직접 준 요소에는 `[hidden]{display:none}`을 짝으로 넣는다. 저자 CSS가 UA의 hidden 처리를 덮어쓴다.
- `filter`가 걸린 조상 아래의 `position:fixed`는 뷰포트가 아니라 그 조상 기준이 된다. 플로팅 요소는 `.machine` 바깥에 둔다.
- `scrollTo({behavior:'auto'})`는 CSS `scroll-behavior:smooth`를 따른다. 즉시 이동은 `'instant'`.
- flex-wrap에서 `flex:1` 아이템은 줄바꿈 전에 수축한다. 줄바꿈을 원하면 basis로 강제한다.
- 픽셀 폰트는 역할 분리: 작은 라벨은 Silkscreen, 큰 숫자는 Press Start 2P (Silkscreen의 4는 큰 크기에서 뭉개진다).
- 검색 매칭 대상은 name+desc+badge+owner 전부 유지한다. 사용자는 브랜드로 검색한다.
